## 1. Способ деплоя

Бот развёрнут на Raspberry Pi 4 (8 ГБ RAM). Это одноплатный компьютер, который работает 24/7, потребляет мало электричества и не требует никаких облачных сервисов — всё работает локально.

На Raspberry Pi запущены:
- **PostgreSQL 16** в Docker-контейнере — основная база данных (~80 МБ RAM)
- **Redis 7** в Docker-контейнере — кэш (~20 МБ RAM)
- **Telegram-бот** — Grammy + Node.js 18, запущен через systemd (~80 МБ RAM)

Общее потребление: около 180 МБ из 8 ГБ доступной памяти. Raspberry Pi справляется с запасом.

Скраперы (worker) с Playwright запускаются отдельно на основном ПК, потому что Chromium на ARM потребляет ~400 МБ RAM. Бот работает и без свежих данных скраперов — просто показывает последние сохранённые значения.

## 2. Процесс деплоя

Пошаговая инструкция, которую мы выполнили:

1. Установили Raspberry Pi OS 64-bit и обновили систему
2. Установили Docker для запуска базы данных и Redis
3. Установили Node.js 18 и pnpm (менеджер пакетов)
4. Склонировали проект с GitHub: `git clone https://github.com/DecayDecay/2025-2026-chatbots-u4125-vasilev_a.git`
5. Создали файл `.env` с токеном бота и строкой подключения к БД
6. Запустили PostgreSQL и Redis через `docker compose up -d`
7. Установили зависимости (`pnpm install`) и применили миграции базы данных
8. Запустили бота и проверили что всё работает
9. Настроили systemd-сервис для автозапуска бота при включении Raspberry Pi

Также написан скрипт `deploy/rpi-setup.sh`, который автоматизирует все шаги — достаточно запустить одну команду.

## 3. Автозапуск через systemd

Чтобы бот работал постоянно и запускался после перезагрузки, мы создали systemd-сервис:

- **Файл:** `/etc/systemd/system/sbox-bot.service`
- **Тип:** simple — бот запускается как обычный процесс
- **Restart:** always — если бот упадёт, systemd автоматически перезапустит его через 10 секунд
- **WantedBy:** multi-user.target — бот запускается при каждой загрузке системы

Управление:
- `sudo systemctl status sbox-bot` — посмотреть статус
- `sudo journalctl -u sbox-bot -f` — посмотреть логи в реальном времени
- `sudo systemctl restart sbox-bot` — перезапустить бота

## 4. Проблемы при деплое

### a) node-canvas на ARM
Библиотека node-canvas (для генерации графиков) требует нативных зависимостей — cairo, pango, libjpeg. На Raspberry Pi их нужно устанавливать отдельно через apt. Без них `npm install` падает с ошибкой компиляции.

Решение: добавили в скрипт установку всех нужных библиотек: `libcairo2-dev`, `libjpeg-dev`, `libpango1.0-dev`, `libgif-dev`, `librsvg2-dev`.

### b) Prisma на Windows vs Linux
На Windows Prisma генерирует DLL-файл для работы с базой. Если перенести проект на Linux — этот файл не подходит. Нужно заново запустить `prisma generate` на целевой платформе.

Решение: `node_modules` не копируется — делаем `pnpm install` и `prisma generate` уже на Raspberry Pi.

### c) Playwright на ARM
Playwright (для скрапинга sbox.game) на ARM-процессоре Raspberry Pi потребляет ~400 МБ RAM и работает заметно медленнее. Для бота с 8 ГБ это не критично, но создаёт лишнюю нагрузку.

Решение: вынесли worker со скраперами на основной ПК. На Raspberry Pi работает только бот + база данных.

## 5. Сбор обратной связи

В бот встроена команда `/feedback` (также доступна через кнопку «Отзыв» в главном меню):

1. Пользователь нажимает /feedback или кнопку «Отзыв»
2. Появляются 5 inline-кнопок с оценками от 1 до 5 звёзд
3. После выбора оценки бот предлагает написать текстовый комментарий (необязательно)
4. Отзыв сохраняется в таблицу Feedback с полями: userId, username, rating, text, дата

Такой формат удобен — пользователю не нужно ничего печатать, достаточно нажать одну кнопку. Текст пишут те, кто хочет дать развёрнутый фидбэк.

## 6. Улучшения на основе обратной связи

В процессе тестирования пользователи предложили ряд улучшений, которые мы реализовали:

### a) Кнопки вместо команд
Главная просьба: не хочется запоминать и печатать команды. Переделали весь интерфейс на inline-кнопки. Теперь при /start открывается меню из 10 кнопок, и из любого раздела можно вернуться назад одной кнопкой «← Меню».

### b) Удаление старых сообщений
Пользователи жаловались что чат засоряется при навигации между разделами. Добавили автоматическое удаление предыдущего сообщения бота при переходе к новому разделу.

### c) Настройки валюты
Просьба: отображать цены в тенге (KZT) вместо долларов. Добавили раздел «Настройки» с выбором валюты. Курс обновляется автоматически. Все цены пересчитываются при отображении — в базе хранятся только доллары.

### d) Ручное добавление в портфолио
Раньше можно было только импортировать из Steam. Добавили пошаговый флоу: бот спрашивает название предмета → количество → цену покупки. Поддерживает ввод цены с пробелами (7 400 = 7400).

### e) Редактирование позиций
Добавили кнопки для каждой позиции в портфолио: «Редактировать цену» и «Удалить». Раньше удалить позицию было нельзя.

### f) Иконки предметов
По просьбе пользователей добавили картинки предметов из Steam CDN в карточку предмета. Теперь при просмотре видно как выглядит скин.

Бот успешно развёрнут на Raspberry Pi 4 и работает 24/7. Systemd обеспечивает автоматический перезапуск при сбоях и после перезагрузки.

Бот работает как полноценный инвестиционный терминал для рынка s&box, доступный с любого устройства через Telegram.
