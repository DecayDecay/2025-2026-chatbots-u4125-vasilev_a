## 1. Описание задачи

Для лабораторной работы выбран вариант «Информационный агрегатор». Идея — сделать Telegram-бота, который собирает в одном месте все данные о рынке скинов для игры s&box в Steam (appid 590830).

Зачем это нужно: в Steam Community Market нет удобного способа следить за всеми предметами сразу. Чтобы посмотреть цены, нужно заходить в каждую карточку по отдельности — это 78 предметов на 8 страницах. Наш бот решает эту проблему — показывает всё в одном месте с аналитикой.

Основные функции бота:
- Мониторинг цен всех 78 скинов в реальном времени
- Top Movers — топ-20 предметов по изменению цены за сутки
- Hot — топ-15 самых продаваемых предметов за 30 дней
- Blue Chips — топ-10 по выручке за всё время
- Портфолио — учёт своих покупок с расчётом прибыли (с учётом 13% комиссии Steam)
- Импорт инвентаря из Steam — автоматическая загрузка предметов из профиля
- Алерты — уведомления когда цена достигает заданного значения
- Графики цен — PNG-картинки за последние 30 дней
- Watchlist — список избранных предметов
- Настройки валюты — отображение в долларах (USD) или тенге (KZT)

## 2. Промпты для LLM

### Начальный промпт

Разработка велась с помощью Claude Code (AI-ассистент). Начальный промпт:

> «Создай Telegram-бота на фреймворке Grammy + TypeScript для мониторинга рынка скинов s&box в Steam. Бот должен подключаться к PostgreSQL через Prisma ORM, показывать цены, объёмы торгов, order book, управлять портфолио и алертами. Используй inline-клавиатуры для навигации.»

### Итерации и доработки

- **Итерация 1:** Проблемы с форматированием — Telegram требует экранирования спецсимволов в MarkdownV2. Написали функцию `esc()` для автоматического экранирования.
- **Итерация 2:** Добавили интерактивное меню — 10 inline-кнопок в /start вместо ручного ввода команд. Так намного удобнее.
- **Итерация 3:** Не работала библиотека для графиков (chartjs-node-canvas). Заменили на чистый node-canvas и рисуем графики вручную — получилось даже красивее.
- **Итерация 4:** Добавили поддержку валют (USD/KZT) — все цены хранятся в долларах, при показе конвертируются по актуальному курсу.
- **Итерация 5:** Переделали навигацию — бот удаляет старые сообщения при переходе между разделами, чтобы чат не засорялся.

## 3. Технологический стек

- **Grammy** — фреймворк для Telegram-ботов на TypeScript
- **TypeScript + tsx** — типизированный код, запуск без компиляции
- **Prisma ORM + PostgreSQL** — база данных с автоматическими миграциями
- **node-canvas** — рисование графиков на сервере (PNG-картинки)
- **Steam Community Market API** — получение цен и объёмов (без авторизации)
- **Docker Compose** — запуск PostgreSQL и Redis в контейнерах
- **pnpm workspaces** — монорепозиторий (бот, веб-дашборд, скраперы в одном проекте)

Проект устроен как монорепа: `apps/bot` (Telegram-бот), `apps/web` (веб-дашборд на Next.js), `apps/worker` (скраперы данных), `packages/db` (схема базы данных).

## 4. Функции бота

Бот @sboxterminal_bot доступен в Telegram. При запуске (`/start`) показывает приветствие с картинкой, статистику рынка и меню из 10 кнопок:

### Top Movers
Показывает топ-20 предметов по изменению цены за 24 часа. Зелёная стрелка — цена выросла, красная — упала. Для каждого предмета показаны: цена, изменение в процентах, объём торгов. Кнопки: график и ссылка на Steam для топ-5.

### Hot (Горячие)
Топ-15 предметов по количеству продаж за 30 дней. Данные берутся с сайта sbox.game. Медали для первых трёх мест. Всегда показывает цены в долларах.

### Blue Chips
Топ-10 предметов по общей выручке за всё время. Это самые «надёжные» скины рынка — аналог голубых фишек на бирже.

### Watchlist (Избранное)
Список предметов, которые пользователь добавил в избранное. Можно добавлять прямо из карточки предмета одной кнопкой.

### Портфолио
Управление своими позициями. Можно добавить предмет вручную (название → количество → цена покупки), импортировать из Steam-инвентаря или удалить. Считает общую стоимость и прибыль с учётом 13% комиссии Steam. Поддерживает ввод цены в KZT.

### Импорт из Steam
Два способа: автоматический (по ссылке на профиль Steam) или ручной (через консоль браузера). При автоматическом способе бот сам загружает инвентарь и создаёт позиции. Дубликаты пропускаются.

### Алерты
Уведомления о цене. Выбираешь предмет, задаёшь порог — бот пришлёт сообщение когда цена станет выше или ниже заданного значения.

### Графики
PNG-картинка с графиком цены за последние 30 снапшотов. Оранжевая линия на тёмном фоне, подписи цен и дат.

### Настройки
Выбор валюты отображения: USD или KZT. Текущая валюта отмечена галочкой. Все цены пересчитываются по актуальному курсу.

### Обратная связь
Оценка бота от 1 до 5 через inline-кнопки + необязательный текстовый комментарий. Сохраняется в базу данных.

## 5. Проблемы и решения

### MarkdownV2
Telegram требует экранирования кучи спецсимволов (`_`, `*`, `[`, `]`, `|`, `.` и другие). Если забыть — сообщение не отправится. Написали функцию `esc()`, которая автоматически добавляет обратные слэши перед всеми спецсимволами.

### Библиотека для графиков не работала
chartjs-node-canvas v4 оказалась несовместима с chart.js v4 — вылетала ошибка при запуске. Вместо того чтобы искать обходной путь, просто заменили на чистый node-canvas и нарисовали графики вручную. Получилось даже лучше — полный контроль над внешним видом.

### ESM-модули и __dirname
В новых модулях JavaScript (ESM) нет привычной переменной `__dirname`. Пришлось заменить её на конструкцию с `import.meta.url` — это стандартный способ получить путь к файлу в ESM.

### Node 18 и fetch
В Node.js 18 встроенная функция `fetch` работает нестабильно — падает при загрузке больших ответов. Заменили на стандартный модуль `https`, который работает без проблем.

## 6. Выводы

Создан полноценный Telegram-бот для мониторинга рынка скинов s&box. Бот собирает данные из Steam и sbox.game, показывает аналитику, управляет портфолио и алертами — всё через удобное меню с кнопками.

Главный вывод: пользователям удобнее работать через кнопки, чем вводить команды вручную. Поэтому весь интерфейс построен на inline-клавиатурах с навигацией.
